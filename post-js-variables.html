JavaScript Variables: var vs let vs const

S
By Saman· 6 min read· June 2026

When you start learning JavaScript, one of the first questions you will have is: should I use var, let, or const? All three are used to store data, but they work differently. Let us understand each one clearly.

Advertisement

What is a Variable?

A variable is like a box where you store information. You give the box a name, and you can put something inside it or change what is inside later.

let name = "Saman";
console.log(name); // Output: Saman

var — The Old Way

var was the original way to declare variables in JavaScript. However, it has some problems that confused many developers.

var age = 20;
var age = 25; // No error! var allows re-declaration
console.log(age); // Output: 25

The problem with var is that it can be re-declared, which can cause unexpected bugs in your code.

let — The Modern Way

let was introduced in ES6 (2015) and is now the standard way to declare variables that will change.

let score = 10;
score = 20; // OK — you can change the value

let score = 30; // ERROR! Cannot re-declare with let
💡 Use let when the value of your variable will change — like a score, counter, or user input.

const — For Values That Never Change

const is used when you want to store a value that should never change. Once you set it, it stays the same forever.

const siteName = "DebugDiaries";
siteName = "Other"; // ERROR! Cannot reassign a const
💡 Use const when the value should not change — like a website name, API key, or fixed number.

Simple Rule to Remember

  • Use const by default
  • Use let only if the value will change
  • Never use var in modern JavaScript
